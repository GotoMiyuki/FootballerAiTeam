# 全链路排查与修复：子 Agent 任务指令未被正确使用，导致输出模板化

## 背景与问题
在 FootballAI 多智能体系统中，用户输入不同问题时，Manager 已经能正确路由到对应的子 Agent（如 `Career`、`Coach`）。然而，即使路由正确，子 Agent 输出的仍然是其固定模板内容（例如：`Career` 总是输出“职业发展路径对比”，`Coach` 总是输出“训练计划”），没有对用户的具体问题做出针对性回应（如公关声明、战术适配分析等）。这导致系统虽然路由灵活，但输出内容僵硬。

## 根因假设
1. **`sub_task` 未正确传递**：Manager 生成的 `sub_tasks` 可能没有被有效注入到子 Agent 的上下文中。
2. **子 Agent 的 System Prompt 固定**：子 Agent 的 System Prompt 没有包含“根据 Manager 下发任务指令生成内容”的规则，导致它们总是按照预设模板输出。
3. **`get_context` 方法未考虑任务指令**：`BaseAgent.get_context` 方法只拼接历史对话，没有优先使用 `current_task`，导致子 Agent 仍然受到前几轮对话影响。

## 排查步骤
请检查以下文件，并回答确认：
1. **`agents/base.py`**：`get_context(self, messages, current_task=None)` 方法是否支持接收 `current_task` 参数？如果支持，它是否优先使用 `current_task` 作为上下文？
2. **`agents/manager.py`**：`_analyze_intent_and_plan` 是否生成了 `sub_tasks` 字段（包含每个 Agent 的具体任务描述）？
3. **所有子 Agent 的 `run` 方法**（如 `career.py`、`coach.py`、`nutrition.py`、`analyst.py`、`document.py`）：它们是否从 `state["agent_outputs"]["_manager_plan"]["sub_tasks"]` 中提取了对应自己的任务指令？是否将它传给了 `get_context` 或直接用于 Prompt？
4. **所有子 Agent 的 System Prompt**：是否包含“根据下发的具体任务指令生成内容”的明确规则？

## 修复方案（请严格按以下步骤执行）

### 1. 改造 `BaseAgent.get_context`
修改 `agents/base.py`，确保 `get_context` 方法支持 `current_task` 参数，并且**如果传入了 `current_task`，则完全忽略历史对话 `messages`，只使用 `current_task` 作为上下文**，避免历史记忆干扰。

### 2. 改造所有子 Agent 的 `run` 方法
对于每一个子 Agent（Career、Coach、Nutrition、Analyst、Document）：
- 在 `run` 函数中，从 `state` 中提取 `sub_task`：
  ```python
  plan = state.get("agent_outputs", {}).get("_manager_plan", {})
  sub_task = plan.get("sub_tasks", {}).get(self.name, "")
调用 get_context 时，传入 current_task=sub_task。

如果 sub_task 为空，则退回到原逻辑（使用最近消息）。

3. 修改所有子 Agent 的 System Prompt
在每个子 Agent 的 System Prompt 中加入以下核心指令（可写在开头或结尾）：

“你必须根据 Manager 下发的具体任务指令来生成输出，不要使用预设模板。任务指令如下：{task_instruction}。请基于此指令，并结合球员数据，输出针对性报告。”
（请确保每个 Agent 都包含此指令，且 task_instruction 从上下文动态填充。）

4. 确保 Manager 正确生成 sub_tasks
检查 manager.py 中的 _analyze_intent_and_plan，确保它返回的 JSON 包含 sub_tasks 字段，且字段的 key 是 Agent 名称，value 是具体任务描述。

5. 调整 Document Agent 的职责
确保 Document Agent 接收到的数据只包含实际调用的 Agent 输出，不要额外生成缺失部分的章节。

验证测试
完成上述修改后，请分别测试以下场景，观察输出是否针对性强且无模板痕迹：

测试1：输入“我想了解转会去皇家马德里的可能性，帮我分析一下战术适配性” → 应输出关于皇马战术适配的具体分析，而不是通用训练计划。

测试2：输入“我和校园女明星一夜情之后流传出花边新闻，应该怎么办？” → 应输出公关声明与职业专注建议，而不是职业发展路径对比。
